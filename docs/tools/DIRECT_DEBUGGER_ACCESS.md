# 🔧 Chrome DevTools 직접 접속 가이드

## ⚠️ 개발자 메뉴가 작동하지 않을 때

"Show developer action button"을 클릭했는데 흰 화면만 나오는 경우, **Chrome DevTools를 직접 브라우저에서 열 수 있습니다!**

---

## 🚀 직접 접속 방법 (가장 빠름!)

### 1단계: Chrome 브라우저 열기

PC에서 Chrome 브라우저를 엽니다.

### 2단계: 주소창에 URL 입력

주소창에 다음을 입력하고 Enter:

```
http://localhost:8081/debugger-ui
```

### 3단계: Console 탭 확인

Chrome DevTools가 열리면:
1. **Console 탭** 클릭
2. 여기에 **모든 로그**가 표시됩니다!

---

## ✅ 확인할 수 있는 것

### 로그 확인
- `console.log()` 출력
- `console.error()` 출력
- 에러 메시지
- 스택 트레이스

### 디버깅 기능
- 브레이크포인트 설정
- 변수 값 확인
- 네트워크 모니터링

---

## 🔧 문제 해결

### Chrome DevTools가 흰 화면인 경우

1. **새로고침** (F5)
2. **앱 재로드** (터미널에서 `r` 키)
3. **다시 접속:**
   ```
   http://localhost:8081/debugger-ui
   ```

### 연결이 안 되는 경우

1. **Metro 번들러가 실행 중인지 확인** (`npm start`)
2. **포트 확인** (8081 포트가 열려 있는지)
3. **방화벽 확인** (8081 포트가 차단되지 않았는지)

---

## 📊 대안: 터미널에서 로그 확인

디버거 연결 없이 터미널에서 로그를 확인하고 싶다면:

1. **디버거 연결 해제** (이미 연결되어 있다면)
2. **Metro 번들러 터미널 확인**
3. 여기에 로그가 표시됩니다

---

## 💡 팁

### 빠른 접속
- Chrome 브라우저 북마크에 추가:
  ```
  http://localhost:8081/debugger-ui
  ```

### 자동 새로고침
- Chrome DevTools에서 **Network 탭** → **Disable cache** 체크
- 앱이 변경되면 자동으로 반영됩니다
