# 🔍 디버거 없이 로그 확인하기

## ⚠️ 문제 상황

`http://localhost:8081/debugger-ui`에 접속했는데 **JSON만 나오는 경우**:
- 이것은 **Expo manifest 정보**입니다
- 디버거가 **연결되지 않았다**는 의미입니다
- Chrome DevTools가 열리지 않습니다

---

## ✅ 해결 방법: 터미널에서 로그 확인

### 방법 1: Metro 번들러 터미널 확인 (가장 간단!)

**디버거가 연결되지 않은 상태에서는:**
- 모든 `console.log()`가 **Metro 번들러 터미널**에 표시됩니다!

**확인 방법:**
1. `npm start` 실행한 터미널 창 확인
2. 여기에 모든 로그가 표시됩니다!

---

### 방법 2: 앱 화면에서 에러 확인

**ErrorBoundary가 에러를 캐치하면:**
- 앱 화면에 **에러 메시지**가 표시됩니다
- "Copy Error" 버튼으로 에러 복사 가능

**확인 방법:**
1. 앱 화면 확인
2. 에러 메시지가 보이면 "Copy Error" 클릭
3. PC에서 Ctrl+V로 붙여넣기

---

### 방법 3: 앱 재로드 후 터미널 확인

**앱을 재로드하면 로그가 터미널에 표시됩니다:**

1. **터미널에서 `r` 키** 누르기 (앱 재로드)
2. **터미널 확인** - 로그가 나타납니다!

---

## 🎯 실전 가이드

### 시나리오 1: 흰 화면이 보일 때

**확인 순서:**
1. ✅ **터미널 확인** - 에러 메시지가 있는지 확인
2. ✅ **앱 화면 확인** - ErrorBoundary 에러 화면이 있는지 확인
3. ✅ **앱 재로드** - 터미널에서 `r` 키

---

### 시나리오 2: 로그가 안 보일 때

**확인 사항:**
1. ✅ **Metro 번들러가 실행 중인지** 확인 (`npm start`)
2. ✅ **디버거가 연결되어 있는지** 확인
   - 연결되어 있으면: Chrome DevTools Console 확인
   - 연결 안 되어 있으면: Metro 번들러 터미널 확인
3. ✅ **앱 재로드** - 터미널에서 `r` 키

---

| 에러 발생 | ErrorBoundary 화면 + 터미널 |

## 🔧 디버거 연결 방법 (다시 시도)

### 방법 1: 터미널에서 개발자 메뉴 열기

1. **Metro 번들러 터미널**에서 `m` 키 누르기
2. 개발자 메뉴가 기기에서 열림
3. "Show developer action button" 클릭
4. "Debug Remote JS" 선택

### 방법 2: 기기에서 직접

1. **기기 흔들기** (Shake gesture)
2. 개발자 메뉴 열림
3. "Debug Remote JS" 선택
