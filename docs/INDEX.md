## 🔍 문서 검색 팁

### 문제가 발생했을 때
1. **에러 메시지가 있다면** → `troubleshooting/` 폴더 확인
2. **디버깅이 필요하다면** → `debugging/` 폴더 확인
3. **도구 사용법이 궁금하다면** → `tools/` 폴더 확인

### 처음 시작할 때
1. `getting-started/QUICK_START.md` 먼저 읽기
2. `getting-started/PROJECT_INITIALIZATION.md` 확인
3. `architecture/` 폴더에서 전체 구조 파악

### 개발 중일 때
1. `architecture/` 폴더에서 설계 원칙 확인
2. `debugging/` 폴더에서 디버깅 방법 확인
3. 문제 발생 시 `troubleshooting/` 폴더 참고
