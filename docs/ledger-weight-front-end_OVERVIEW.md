# 📱 '장부의 무게' 프로젝트 프론트엔드 개요

## 📋 프로젝트 개요

- **프로젝트명**: 장부의 무게
- **개발 스택**: React Native (TypeScript)
- **레포지토리**: ledger-weight-front-end
- **통신 방식**: WebSocket (실시간 양방향 통신)
- **상태 관리**: Redux Toolkit
- **목표**: WebSocket 기반 실시간 카드 게임 클라이언트 애플리케이션 개발

## 🎯 핵심 기능

### 통신 프로토콜
- **수신**: `GAME_STATE_UPDATE` - 게임 상태 업데이트 수신
- **송신**: `PLAYER_ACTION` - 플레이어 액션 전송

### 게임 상태 관리
- 복잡한 게임 상태 관리
- 플레이어 핸드 카드 관리
- 실시간 게임 보드 상태 동기화

## 👑 게임 명칭 체계 (UI 시각화 대상)

### 역할 및 기본 정보

| 구분 | 명칭 | UI 노출 위치 |
|------|------|--------------|
| **플레이어 역할** | 상단주, 원로원, 적도 세력, 야망가 | 플레이어 목록 및 인게임 플레이어 목록 |
| **재력/영향력** | 재력 (HP), 영향력 (사거리) | 플레이어 아바타 옆/상단 |
| **장착 보물** | 협상 증표, 비밀 장부, 연속 상환 요구 등 16종 | 플레이어 목록 및 본인 캐릭터 영역 |

### 카드 시스템

| 구분 | 명칭 | 기능 설명 |
|------|------|-----------|
| **무늬** | 검, 책, 치유, 돈 | 카드 디자인의 4가지 아이콘 요소 |
| **숫자** | 상, 대, 중, 소 | 카드 디자인의 A, K, Q, J 위치에 표시 |
| **핵심 카드** | 정산, 회피, 비상금, 전원 견제, 승부, 강제 압류, 영업 금지 등 | 카드 UI 텍스트 및 효과 설명 |

## 🔑 WebSocket 프로토콜 기반 UI 설계

### 수신 (GAME_STATE_UPDATE) 기반 UI

#### 게임 보드 영역
- 모든 플레이어의 상태 표시
  - 재력 (HP)
  - 영향력 (사거리)
  - 장착 보물
  - 테이블 카드
- 원형 또는 테이블 형태로 레이아웃 구성

#### 턴 상태
- 현재 턴 플레이어 강조 표시
- 제한 시간 카운트다운 (`turn_state.time_left`)

#### 액션/로그
- `event` 필드를 통한 게임 로그 표시
- 알림 메시지 (예: "P003이 P005에게 정산을 사용했습니다.")

### 송신 (PLAYER_ACTION) 기반 인터페이스

#### 핸드 카드
- 플레이어가 손에 든 카드 표시
- 카드 탭/드래그로 `USE_CARD` 액션 요청

#### 대상 선택 (타겟팅)
- 카드 사용 시 영향력 범위 내 플레이어 선택
- 타겟팅 UI 구현

#### 응답 인터페이스
- `turn_state.required_response`가 있을 때
- '회피 사용' 또는 '포기' 등의 `RESPOND_ATTACK` 버튼
- 팝업 형태로 표시

#### 턴 종료
- 턴 종료 버튼 (`END_TURN` 요청)
