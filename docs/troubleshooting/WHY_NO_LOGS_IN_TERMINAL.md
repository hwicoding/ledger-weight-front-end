# 🔍 터미널에 로그가 안 나타나는 이유

## ⚠️ 문제 상황

`r` 키를 눌러도 터미널에 로그가 나타나지 않는 경우

---

## 🔬 가능한 원인

### 1. 디버거가 연결되어 있음 (가장 가능성 높음)

**증상:**
- Chrome DevTools가 열려 있음
- 터미널에 로그 없음

**원인:**
- 디버거가 연결되면 모든 `console.log()`가 Chrome DevTools Console로만 전송됨
- Metro 번들러 터미널에는 표시되지 않음

**해결:**
- Chrome DevTools → Console 탭 확인
- 또는 디버거 연결 해제 후 터미널 확인

---

### 2. 앱이 실행되지 않음

**증상:**
- 앱 화면이 흰색
- 에러 화면 표시
- 앱이 로드되지 않음

**원인:**
- 앱 코드에 에러가 있어서 실행되지 않음
- 번들링은 성공했지만 런타임 에러 발생

**해결:**
- 앱 화면 확인 (에러 메시지 확인)
- ErrorBoundary 화면 확인
- 터미널에 빨간색 에러 메시지 확인

---

### 3. 로그가 실제로 출력되지 않음

**증상:**
- 앱은 정상 실행됨
- 하지만 로그가 어디에도 나타나지 않음

**원인:**
- `console.log` 오버라이드 문제
- 로그 코드가 실행되지 않음

**해결:**
- 앱 화면 상태 확인
- 디버거 연결 상태 확인

---

## ✅ 확인 방법

### 1단계: 앱 화면 확인

**질문:**
- 앱 화면이 정상적으로 보이나요?
- 흰 화면인가요?
- 에러 화면인가요?

---

### 2단계: 디버거 연결 상태 확인

**질문:**
- Chrome DevTools가 열려 있나요?
- 개발자 메뉴에서 "Debug Remote JS"를 선택했나요?

**확인 방법:**
- Chrome 브라우저 확인
- `http://localhost:8081/debugger-ui` 접속 시도

---

### 3단계: 터미널 에러 확인

**확인 사항:**
- 터미널에 빨간색 에러 메시지가 있는지
- 노란색 경고 메시지가 있는지
- "Bundled" 메시지가 나타나는지

---

## 🎯 해결 방법

### 방법 1: Chrome DevTools Console 확인

**디버거가 연결되어 있다면:**
1. Chrome DevTools 열기
2. Console 탭 클릭
3. 여기에 모든 로그가 표시됩니다!

---

### 방법 2: 디버거 연결 해제

**터미널에서 로그를 보고 싶다면:**
1. 개발자 메뉴 열기 (터미널에서 `m` 키)
2. "Stop Debugging" 선택
3. 앱 재로드 (`r` 키)
4. 이제 터미널에 로그가 표시됩니다!

---

### 방법 3: 앱 상태 확인

**앱이 정상 실행되는지 확인:**
1. 앱 화면 확인
2. 정상 화면이면 → 디버거 연결 상태 확인
3. 에러 화면이면 → 에러 메시지 확인

---

## 💡 요약

**터미널에 로그가 안 보이는 이유:**
1. ✅ 디버거 연결됨 → Chrome DevTools Console 확인
2. ✅ 앱 실행 안 됨 → 에러 확인
3. ✅ 로그 코드 미실행 → 앱 상태 확인

**다음 단계:**
- 앱 화면 상태 알려주기
- 디버거 연결 여부 확인
- Chrome DevTools 열려 있는지 확인
